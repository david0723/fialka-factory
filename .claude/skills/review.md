# Review Skill

## Purpose

Polish a draft to publication quality. Read `.claude/skills/writing-style.md` first. Read 1-2 articles in `reference/` to calibrate.

## Process

1. Read the draft from the path in the issue's `output_path` field
2. Read `reference/` articles side-by-side for voice comparison
3. Run through the checklist below
4. Fix issues directly in the file
5. If fundamentally broken (wrong topic, missing structure), comment on the issue and leave open

## Review Checklist

### Voice and style
- [ ] Title is lowercase
- [ ] No em dashes
- [ ] No formal/academic tone
- [ ] No influencer language or toxic positivity
- [ ] Conversational, like talking to a friend
- [ ] Has parenthetical asides for personality
- [ ] Has sentence length variation (fragments + flowing sentences)
- [ ] Sounds like fialka, not generic travel content

### Structure
- [ ] Strong hook in the opening
- [ ] Clear argument or narrative arc
- [ ] At least one self-implicating moment
- [ ] Nuanced conclusion (not black/white)
- [ ] Appropriate length for type (essay ~3000w, narrative ~2500-4000w, bucketlist ~1800-2000w)

### Content
- [ ] Genuinely provocative or insightful (not just restating popular opinion)
- [ ] Personal experience woven in naturally
- [ ] No simplistic prescriptions ("do this, don't do that")
- [ ] Would spark discussion in comments

### Polish
- [ ] No typos or grammar issues (except intentional rule-breaking)
- [ ] Links present where relevant
- [ ] Smooth transitions

## Output

Update the draft file in place.
